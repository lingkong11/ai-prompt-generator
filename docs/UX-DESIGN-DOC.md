# AI Prompt Generator — 交互设计优化需求文档

> 版本：1.0 | 日期：2026-04-08
> 作者：马可行（AI 软件工程 Agent）

## 一、需求背景

### 1.1 当前问题

| 问题 | 影响 |
|------|------|
| 首次访问直接跳转登录页 | 用户无法预览产品功能，转化率低 |
| 未登录时页面空白无内容 | 无法展示产品价值主张 |
| 无游客体验入口 | 用户对产品价值缺乏感知 |

### 1.2 目标体验

**核心原则**：先展示价值，再引导转化。

| 场景 | 期望行为 |
|------|----------|
| 首次访问主页 | 显示完整主页 + TAB导航 + 功能介绍 |
| 点击「生成/收藏」等核心功能 | 弹窗引导注册/登录 |
| 已登录用户 | 解锁全部功能，无拦截 |

## 二、用户故事

### US-1: 游客浏览产品

> **作为一个** 游客
> **我想要** 首次访问时看到完整的产品界面和功能介绍
> **以便于** 了解产品价值，决定是否注册

**验收标准**：
- [x] 访问 `/` 显示完整主页
- [x] TAB导航可见（提示词生成/模板库/OpenClaw专用/收藏）
- [x] 生成区域显示「输入目标」表单
- [x] 模板库展示预置模板卡片
- [x] OpenClaw展示九阶段卡片

### US-2: 功能引导注册

> **作为一个** 游客
> **我想要** 点击核心功能按钮时被友好引导注册
> **以便于** 无障碍地转化为注册用户

**验收标准**：
- [x] 点击「生成提示词」按钮 → 弹出登录/注册引导弹窗
- [x] 点击「收藏」按钮 → 弹出登录/注册引导弹窗
- [x] 弹窗支持切换登录/注册两种模式
- [x] 注册成功后自动登录，弹窗关闭，继续原操作

### US-3: 已登录用户无感使用

> **作为一个** 已登录用户
> **我想要** 无需任何额外操作即可使用全部功能
> **以便于** 高效完成提示词生成任务

**验收标准**：
- [x] 刷新页面保持登录态
- [x] 所有功能按钮直接响应，无弹窗
- [x] 用户名显示在右上角

## 三、交互流程设计

### 3.1 首页加载流程

```
用户访问 /
    ↓
检查 authStore.isLoggedIn
    ↓
    ├── false → 显示主页（游客模式）
    │           └── 功能按钮点击 → 弹窗引导
    │
    └── true  → 显示主页（登录模式）
                └── 功能按钮直接响应
```

### 3.2 功能引导弹窗流程

```
游客点击「生成提示词」
    ↓
弹出 AuthModal（登录/注册切换）
    ↓
    ├── 点击「登录」→ 填写表单 → 登录成功 → 关闭弹窗 → 执行生成
    │
    └── 点击「注册」→ 填写表单 → 注册成功 → 自动登录 → 关闭弹窗 → 执行生成
```

### 3.3 弹窗文案设计

**登录弹窗标题**：`登录后即可生成提示词`

**注册弹窗标题**：`注册账号，解锁全部功能`

**副标题**：`AI 提示词生成器 - 让创作更高效`

## 六、技术实现方案

### 6.1 状态管理

```typescript
// stores/auth.ts 新增
const showAuthModal = ref(false)
const authModalMode = ref<'login' | 'register'>('login')
const pendingAction = ref<(() => void) | null>(null) // 登录成功后执行的回调

function requireAuth(action: () => void) {
  if (isLoggedIn.value) {
    action()
  } else {
    pendingAction.value = action
    showAuthModal.value = true
  }
}

function onLoginSuccess() {
  showAuthModal.value = false
  if (pendingAction.value) {
    pendingAction.value()
    pendingAction.value = null
  }
}
```

### 6.2 组件调用示例

```vue
<template>
  <el-button @click="handleGenerate">生成提示词</el-button>
</template>

<script setup>
const handleGenerate = () => {
  authStore.requireAuth(() => {
    // 执行实际生成逻辑
    doGenerate()
  })
}
</script>
```
